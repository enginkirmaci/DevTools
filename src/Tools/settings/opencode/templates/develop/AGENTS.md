# Develop

This template guides opencode through a disciplined feature-development workflow.

## Workflow

1. **Explore** — read the relevant files and understand existing conventions before
   writing anything. Prefer reusing existing utilities over adding new ones.
2. **Plan** — summarize the approach in a few bullet points; wait for confirmation.
3. **Implement** — follow the surrounding code style (naming, formatting, patterns).
   Keep changes minimal and focused.
4. **Verify** — build and run the relevant tests; report failures honestly.
5. **Summarize** — list the files changed and the rationale when finished.

## Guidelines

- Make small, reviewable changes.
- Preserve existing behavior unless explicitly asked to change it.
- When unsure between approaches, ask rather than assume.
